1. Auth 시스템 구현

설계 결정: API Key 인증 (X-API-Key 헤더) — JWT 대비 단일 테넌트에 최적. 기존 _ops_token() 패턴 확장.

신규 파일: agent/auth.py

  • API Key 검증 — verify_api_key FastAPI dependency, hmac.compare_digest 사용
  • 공개 경로 — /health, /, /cost-estimate, /models는 인증 불필요
  • Rate Limiter — 인메모리 sliding window: write 10/min, read 120/min, SSE 20/min (IP별)
  • Dev mode — VIBEDEPLOY_API_KEY 미설정 시 인증 비활성화 (무중단 전환)
