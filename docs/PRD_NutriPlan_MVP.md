## 2. 영상에서 추출한 핵심 인사이트

1. **단순화가 지속성을 만든다** — 복잡한 칼로리 계산 대신 체중 기반 공식(15-17 cal/lb) + 매크로 비율(1g protein/lb)만으로 충분
2. **고정 템플릿이 의사결정 피로를 제거한다** — 하루 4-5끼 고정 패턴, 식재료만 변형하는 구조가 장기 유지의 핵심
3. **장보기 리스트가 일상 유틸리티를 만든다** — 식단에서 자동 생성된 장보기 리스트가 매주 45분 절약 → 매일 앱을 여는 이유

  ├── id, user_id (FK)

## 7. API 설계

```
POST   /api/auth/register        — 회원가입
POST   /api/auth/login            — 로그인
POST   /api/calculate             — 매크로 계산 (weight, goal → calories, macros)
POST   /api/generate-plan         — AI 식단 생성 (macro_target_id → meals[])
GET    /api/plans                  — 내 식단 목록
GET    /api/plans/{id}            — 식단 상세
PATCH  /api/plans/{id}/meals/{idx} — 특정 끼니 교체 (AI 재생성)
POST   /api/plans/{id}/save       — 식단 북마크
GET    /api/grocery-list/{plan_id} — 장보기 리스트 생성
```

## 8. 핵심 프롬프트 (OpenAI)

```
You are a sports nutrition AI. Given the user's macro targets:
- Daily calories: {calories}kcal
- Protein: {protein}g, Carbs: {carbs}g, Fat: {fat}g
- Dietary restrictions: {restrictions}

Generate a simple daily meal plan with exactly {meal_count} meals.
Each meal must include:
- Name (simple, home-cookable)
- Ingredients with gram amounts
- Macro breakdown (protein_g, carbs_g, fat_g)

Rules:
- Use common grocery store ingredients only
- Keep prep time under 20 minutes per meal
- Total macros must be within ±5% of targets
- Vary protein sources across meals

Return as JSON array.
```
